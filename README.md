# サーバー崩壊ラプソディ 🖥️💥

**~ Server Crash Rhapsody ~**

ITエンジニアのための、偏差値低め・思想弱め・でもちょっと愛があるクソゲー

## 遊び方

1. **TAP TO START** でゲーム開始
2. サーバーをタップ/クリックして負荷を与える
3. CPU、メモリ、ネットワーク、ディスクの指標が上昇
4. しきい値を超えるとサーバーが**崩壊**
5. 崩壊スコアを確認して **REBOOT SYSTEM**

## 特徴

### 🎲 同じパターンは二度と出ない

- すべてがランダムパラメータの組み合わせで生成
- 崩壊しきい値、負荷増加量、減衰率がプレイごとに変化
- 突然死タップ数、謎のクラッシュ時間も毎回ランダム

### 💥 6種類の崩壊演出

| 演出 | 説明 |
|------|------|
| 爆発 | パーティクルが飛び散る |
| フリーズ | スキャンラインが走って停止 |
| ドット散乱 | サーバーがバラバラに崩れる |
| ブルースクリーン | Windowsユーザーおなじみのアレ |
| 無言消滅 | 静かに消える（一番怖い） |
| グリッチ | 画面がバグる |

### 📝 ランダムログメッセージ

```
[ERROR] Segmentation fault (core dumped)
[ERROR] It works on my machine
[WARN] 嫌な予感がする...
[FATAL] 原因は後で調べます
[PANIC] 誰かー！！
```

### 🎵 8bit サウンド

- Web Audio APIによるレトロ音声生成
- タップ音、警告音、爆発音がすべてプログラム生成
- 無音も演出として採用

### 🎨 レトロスタイル

- 8bit/16bit風ドット絵
- ピクセルフォント（Press Start 2P）
- CRTスキャンライン効果
- 8色制限パレット

## スコア

| 項目 | 説明 |
|------|------|
| 耐久時間 | サーバーが持ちこたえた秒数 |
| 崩壊指標 | 同時に危険域に達した指標の数 (0-4) |
| 芸術点 | 完全ランダム（意味はない） |

## 崩壊トリガー条件

- CPU + メモリ同時超過
- CPU単独暴走
- メモリ単独超過（OOM Killer）
- ネットワーク遅延暴走
- ディスク容量枯渇
- 特定タップ数での突然死
- 謎の時間経過クラッシュ

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript
- Canvas API（サーバー描画）
- Web Audio API（8bit音声生成）
- フレームワーク不使用
